Every assignment in Aourly follows a lifecycle from creation to completion. Understanding this workflow helps you manage your projects effectively, keep clients informed, and ensure nothing falls through the cracks.

## Assignment Statuses

An assignment can be in one of eight statuses. Here is what each one means and when it applies.

### Draft

The starting status for every new assignment. While in Draft:
- You can edit all fields freely (name, pricing, scope, dates, etc.)
- You can add and remove performers
- No time can be logged
- No invoices can be generated

Think of Draft as your workspace for refining the assignment before it goes live.

### Pending Approval

The assignment has been sent to the client for review and sign-off. While in Pending Approval:
- The client has received a link to review the assignment details
- Assignment terms are locked and should not be changed
- The client can approve or reject

> **Warning:** Once sent for approval, assignment terms cannot be changed. If changes are needed, the client must reject, you make edits, and then you send a new version for approval.

### Scheduled

The client has approved the assignment, but the start date is in the future. This status acts as a holding state:
- The approved snapshot has been created
- Performers are assigned and ready
- Time logging is not yet available
- The assignment will automatically transition to Active on the start date

### Active

The main working state. While Active:
- Performers can log time entries
- Expenses can be tracked against the assignment
- Timesheets are generated according to the configured frequency
- Invoices can be created from approved time entries
- Progress tracking is live

### On Hold

A temporary pause state. Use it when:
- The client requests a delay
- You are waiting for external input or approvals
- Budget constraints require a temporary stop
- You need to reallocate resources to other work

Moving to On Hold signals to all performers that work is suspended. No restrictions are enforced on time logging, but the status serves as a clear indicator.

### Completed

All work is finished. Moving to Completed signals that:
- The project deliverables have been met
- Final timesheets should have been approved and invoiced
- No further time entries are expected

### Cancelled

The assignment was terminated before completion. Reasons might include:
- The client cancelled the project
- The scope changed fundamentally, requiring a new assignment
- Business circumstances changed

Cancellation does not delete any data. Existing time entries and expenses remain accessible.

### Rejected

The client reviewed the assignment and declined it. From Rejected, you can:
- Edit the assignment terms
- Send it for approval again with updated conditions
- Cancel it if the engagement is not going forward

## Status Transitions

Not all transitions are available from every status. Here is the flow:

```
Draft --> Pending Approval --> Scheduled --> Active --> Completed
  |            |                             |   ^
  |            v                             |   |
  |         Rejected                     On Hold-+
  |
  +--------------------------------------------> Cancelled
```

**Key transitions:**
- **Draft to Pending Approval**: Send assignment for client review
- **Draft to Active**: Skip approval and start working immediately
- **Pending Approval to Scheduled/Active**: Client approves (Scheduled if future start date, Active otherwise)
- **Pending Approval to Rejected**: Client declines
- **Rejected to Pending Approval**: Revise and resubmit
- **Active to On Hold**: Temporarily pause work
- **On Hold to Active**: Resume work
- **Active to Completed**: All work finished
- **Any status to Cancelled**: Terminate the assignment

## Client Approval Process

The approval process provides a formal agreement between you and your client. Here is how it works in detail.

### Sending for Approval

1. Open the assignment in Draft (or Rejected) status
2. Click **Send for Approval**
3. The client receives a notification with a link to review the assignment

### What the Client Sees

The client is presented with:
- The assignment name and description
- Pricing type and rates
- Scope and timeline
- Any attached documents (terms of service, liability insurance, etc.)

### Client Signs

When the client agrees to the terms, they digitally sign the assignment. This creates an **approved snapshot** (ApprovedAssignment) containing:
- A complete copy of the assignment data at the moment of approval
- Checksums for any attached documents
- A signature hash combining the signer's identity, timestamp, and all data

This snapshot is immutable and can be verified at any time for integrity.

### Multiple Approvals

If you need to change terms on an already-approved assignment, you can send it for approval again. Each approval creates a new versioned snapshot, and the latest one is considered the current agreement. All previous snapshots are retained for audit purposes.

## Changing Status

To change an assignment's status:

1. Open the assignment detail page
2. Use the status action buttons (e.g., **Send for Approval**, **Activate**, **Put On Hold**, **Complete**)
3. Confirm the transition when prompted

Some transitions may require additional input (e.g., sending for approval triggers a notification to the client).

## Best Practices

### Before Activating
- Verify all performers are added with correct hourly rates
- Confirm the scope and timeline are realistic
- Ensure the client information and billing details are correct

### During Active Work
- Monitor progress regularly using the dashboard
- Review and approve time entries promptly
- Keep the assignment description updated if scope evolves

### Before Completing
- Approve all pending time entries
- Invoice all approved timesheets
- Verify all billable expenses have been included
- Review the assignment summary for any unbilled work

### When to Use On Hold vs. Cancelled
- Use **On Hold** when you expect work to resume
- Use **Cancelled** when the engagement is definitively over
- On Hold preserves the expectation of continuation; Cancelled is final


<Warning>Once sent for approval, assignment terms are locked. If changes are needed after approval, you must create a new version and send it for approval again.</Warning>

<Info>The client approval process is optional. You can activate an assignment directly from Draft if formal client sign-off is not needed.</Info>

<Tip>Approved snapshots serve as a contract record. You can access them at any time from the assignment detail page to verify what was agreed upon.</Tip>

<Warning>Completing or cancelling an assignment does not automatically invoice remaining time entries. Make sure to process all outstanding timesheets first.</Warning>

## Frequently Asked Questions

<AccordionGroup>
  <Accordion title="Can I reopen a Completed assignment?">
    Moving an assignment back from Completed to Active is possible but should be done carefully. Any new time entries will need to go through the normal approval and invoicing flow.
  </Accordion>
  <Accordion title="What happens if the client rejects the assignment?">
    The assignment moves to Rejected status. You can edit the terms and send it for approval again. The rejection does not delete any data.
  </Accordion>
  <Accordion title="Can I skip the approval process entirely?">
    Yes. If you do not need formal client sign-off, you can move an assignment directly from Draft to Active. The approval step is optional but recommended for larger engagements.
  </Accordion>
  <Accordion title="What is stored in the approved snapshot?">
    The snapshot captures the assignment details at the moment of approval: name, description, pricing, scope, dates, and any attached documents. It includes a digital signature hash to verify that the data has not been altered.
  </Accordion>
  <Accordion title="Can I edit an Active assignment?">
    Some fields can be updated on an Active assignment, but core terms (pricing type, scope) may require sending a new version for approval if the client needs to agree to the changes.
  </Accordion>
</AccordionGroup>
